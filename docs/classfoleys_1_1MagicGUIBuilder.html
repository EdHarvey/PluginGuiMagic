Detailed Description

template<class AppType>
class foleys::MagicGUIBuilder< AppType >

The MagicGUIBuilder is responsible to recreate the GUI from a single ValueTree.

You can add your own factories to the builder to allow additional components. The template parameter AppType specifies a type, that each components gets a reference of, it is intended to be eithe AudioProcessor, JUCEApplication or your own MainWindow, it's up to you.

Creates the MagicGUIBuilder.

This is responsible to create the GUI elements from the ValueTree. To allow custom Components to be created with a backlink to the containing application. This can be an AudioProcessor or it's descendents. Or it could be your custom MainComponent. In which case you have to template it with your own type. Now you can reference it in the factory method.

Parameters
parentis the component, the GUI will be imbedded into.
appis the model the components might connect to.
magicStatean optional pointer to the MagicProcessorState. This is only useful for AudioProcessors, but will allow automatic connection of parameters.

References MagicBuilder::config, and MagicBuilder::updateStylesheet().
